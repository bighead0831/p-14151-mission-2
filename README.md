# TDD다항식계산기 -t28까지 구현
## 구조
### 메인 메서드
#### run
- 호출하면 while문이 가장 먼저 돌면서 ()연산자가 있는지 검사하고 있으면 선행 계산한다.(재귀적)
- switch문으로 각 연산자 유무에 따라 연산자 기능(계산)을 한다. 계산순서는 * -> +, -이므로 역으로 case를 써준다. +, - -> *
  - 이유: **낮은 우선순위 연산자로 먼저 분리해야 각 항이 깔끔하게 나뉘어지고, 재귀 호출을 통해 높은 우선순위인 '*'이 나중에 처리되는 구조가 된다.**
  - 예시 ("2+3*4")
    - 방식1(*연산 먼저 배치, 이후 +, -연산): [2+3*4] -> [2+3]*[4] => actionMult는 재귀가 없어서 [2+3]에서 parseInt 오류
    - 방식2(+,-연산 먼저 배치, 이후 *연산): [2+3*4] -> [2]+[3*4] => actionAdd는 재귀가 있어서 [3*4]에서 run 호출 -> [3*4] -> [3]*[4] -> actionMult에서 12로 연산하여 반환 -> [2]+[12] -> actionAdd에서 14로 반환 => 14로 최종 결과값 반환
### 검사 메서드
#### chkBoundary
- ()연산자 유무를 확인해서 true/false로 반환한다.
#### chkAction
- +, -, * 연산자 유무를 확인해서 반환하며, 만일 앞서 말한 연산자들이 없으면 "Keep Going"이란 문자열을 반환한다.
  - Keep Going의 기능: +, - 연산 간 재귀적으로 run을 호출할 일이 있는데 이때 검사용으로 사용한다.
### 실행 메서드
#### actionBoundary
- ()연산자 내 계산식을 선행 계산하는 메서드이다.
- ()연산자는 가장 안쪽 ()연산자부터 선행계산한다.
- 논리: 입력받은 명령 문자열에서 가장 마지막으로 등장하는 '('를 open으로, open 이후 처음 등장하는 ')'를 close로 정의 -> open 이전 문자열 + open~close문자열을 계산식으로 계산해서 그 결과값을 다시 문자열로 저장 + close 이후 문자열 -> 이것을 반환한다.
#### actionAdd
- 덧셈 연산하는 메서드이다.
- 논리: 계산식 내 "+"를 기준으로 계산식 분리 -> 각 계산식에 대해 chkAction 검사 -> chkAction이 "Keep Going"이 아닌 경우(run) / "Keep Going"인 경우(Integer.parseInt) -> 결과값 반환
#### actionSub
- 뺄셈 연산하는 메서드이다.
- 논리: 계산식 내 "-"를 기준으로 계산식 분리 -> 각 계산식에 대해 chkAction 검사 -> chkAction이 "Keep Going"이 아닌 경우(run) / "Keep Going"인 경우(Integer.parseInt) -> 결과값 반환
#### actionMult
- 곱셈 연산하는 메서드이다.
- 논리: 계산식 내 "*"를 기준으로 계산식 분리 / 계산 -> 결과값 반환
  - 곱셈식은 재귀 불필요(chkAction 검사 불필요)
  - 어차피 앞서 +,- 계산이 모두 끝났기 때문에 곱셈만 하면 되기 때문이다.
